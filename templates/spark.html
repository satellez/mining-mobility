Apache Spark sobre siniestros viales

PySpark 3.x MLlib Spark SQL

El mismo dataset siniestros_detalle_completo.json (196.152 registros) analizado anteriormente en Orange Data Mining se reprocesa aquí con Apache Spark usando su API Python (PySpark). El objetivo es mostrar las diferencias de paradigma, escala y capacidades entre ambas herramientas.

Donde Orange necesitó una muestra de 3.000 registros para k-Means (limitación del widget Silhouette), Spark ejecuta sobre el conjunto completo. Donde Orange usa widgets visuales, Spark usa un Pipeline declarativo reproducible y versionable como código.

Motivación

Escalabilidad horizontal

Spark distribuye el procesamiento entre múltiples nodos. Lo que en un solo equipo tarda minutos, en un clúster tarda segundos. El mismo script corre en local o en AWS EMR / Databricks sin cambiar una línea.

Pipeline reproducible

La clase Pipeline de MLlib encadena transformaciones y estimadores en un objeto serializable. Se entrena una vez, se guarda, se carga en producción. No hay configuración manual por widget.

SQL nativo sobre DataFrames

Con createOrReplaceTempView() cualquier DataFrame se consulta con SQL estándar: GROUP BY, WINDOW, JOIN. Spark traduce las queries a un plan de ejecución optimizado (Catalyst).

3 · Random Forest sin OHE — importancia limpia 
# Sin OneHotEncoder: trees manejan índices ordinales

asm = VectorAssembler(inputCols=[c+"_idx" for c in cat_cols] + num_cols, ...)
rf  = RandomForestClassifier(numTrees=50, maxDepth=6, seed=42)

model = Pipeline(stages=indexers+[label_ix, asm, rf]).fit(train_df)

# featureImportances alineado directamente con feature_names

4 · K-Means sobre 196k registros con StandardScaler 
scaler = StandardScaler(inputCol="features_raw", outputCol="features_km",
                        withMean=True, withStd=True)

km_pipeline = Pipeline(stages=[*indexers, assembler, scaler, kmeans])

Comparación — Orange Data Mining vs Apache Spark
Aspecto Orange Data Mining Apache Spark
Paradigma GUI visual, drag-and-drop Código Python, API declarativa
Procesamiento Single-node, RAM del equipo Distribuido · escala horizontal
Dataset en clustering 3.000 registros (límite Silhouette widget) 196.152 registros completos
Importancia features No disponible en k-Means RF featureImportances (Gini) alineado con features
Preprocesamiento Widget Preprocess manual Pipeline declarativo, serializable
Consultas ad-hoc Filtros visuales limitados Spark SQL completo (GROUP BY, WINDOW, JOIN)
Reproducibilidad Archivo .ows exportable Scripts Python versionables en git
Integración Standalone Hadoop, Kafka, S3, Delta Lake, Databricks
Inicio < 5 s (GUI) ~20–40 s (JVM + SparkContext)
Curva de aprendizaje Baja — interfaz visual Media-alta — requiere Python + APIs
Producción Limitada a exploración Apto para pipelines de producción

Orange es ideal para exploración rápida, prototipado visual y comunicación de resultados sin escribir código. Su limitación principal es la escala: algunos widgets (como Silhouette para k-Means) imponen topes a la cantidad de registros.

Spark brilla cuando el volumen de datos supera la RAM disponible, cuando se necesita reproducibilidad de pipeline, o cuando el análisis debe integrarse en un sistema de datos más amplio. La inversión en aprendizaje es mayor, pero el resultado es un proceso auditable, escalable y listo para producción.
