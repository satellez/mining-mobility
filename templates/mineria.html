Minería de Datos — Orange

Orange 3.40.0 · Python 3.12

El proceso de minería se ejecutó sobre el dataset siniestros_detalle_completo.json usando Orange Data Mining. El JSON contiene una estructura anidada (hipótesis, vehículos y actores como arrays por siniestro) que fue aplanada mediante un script Python antes de alimentar los algoritmos.

Se aplicaron dos técnicas complementarias: una supervisada para predecir la gravedad de cada siniestro, y una no supervisada para descubrir patrones latentes de accidentalidad sin usar etiquetas previas.

Algoritmos aplicados
Supervisado

Random Forest

Objetivo: clasificar la gravedad del siniestro (Con Muertos, Con Heridos, Solo Daños) a partir de variables del evento.

Variables de entrada
  • hora — hora del día (0–23)
  • clase_siniestro — tipo de accidente
  • localidad — código de localidad
  • hipotesis — causa principal registrada
  • tipo_vehiculo — primer vehículo involucrado
  • num_vehiculos y num_actores
Árboles100 Evaluación5-fold CV MétricaAUC / Matriz de confusión / ROC
No supervisado

k-Means Clustering

Objetivo: agrupar siniestros en clusters según sus características sin usar la etiqueta de gravedad, para descubrir perfiles de accidentalidad recurrentes.

Variables usadas
  • Las mismas 7 variables de entrada que Random Forest
  • La variable gravedad no interviene en el agrupamiento

Análisis de resultados
Random Forest — Clasificación AUC 0.897

El modelo clasifica correctamente el 83.6% de los 196.152 siniestros. El AUC de 0.897 indica buena capacidad discriminatoria general. El balanceo de clases triplicó el recall de Con Muertos: de 174 aciertos (5.8%) a 506 aciertos (16.9%), a costa de bajar la accuracy global ~3 puntos.

AUC0.897
CA0.836
F10.839
MCC0.647
Punto crítico — Con Muertos Recall 16.9%

Con Muertos representa solo el 1.5% del dataset (2.995 de 196.152 registros). Aunque el balanceo mejoró su recall de 5.8% a 16.9%, el modelo sigue clasificando incorrectamente el 83% de los accidentes fatales. Esto refleja que las variables disponibles (hora, clase, localidad, hipótesis, vehículo) no capturan completamente los factores que determinan la fatalidad de un siniestro.

k-Means — Clustering k=2 óptimo

El algoritmo identifica dos grandes grupos de siniestros con Silhouette = 0.226. Los scores negativos para k≥5 confirman que los datos no soportan más de 2 clusters con estas variables. El Box Plot revela que Solo Daños tiene el Silhouette más alto (0.590) y más estable, mientras que Con Muertos tiene el menor (0.511) con mayor dispersión, indicando que los accidentes fatales son los más heterogéneos y difíciles de agrupar.

k óptimo2

Conclusión general 196.152 siniestros

Con 7 variables derivadas del JSON original, el modelo predice la gravedad con precisión razonable para Con Heridos y Solo Daños, pero limitada para Con Muertos por su escasez en el dataset. Para mejorar: agregar variables temporales (día de semana, mes, año), características de la vía o condiciones climáticas, y evaluar técnicas de oversampling como SMOTE sobre la clase minoritaria.
